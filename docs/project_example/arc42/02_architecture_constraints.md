# 2. Contraintes d'Architecture

Les contraintes qui limitent la liberté de choix de l'architecte lors de la conception.

## 2.1 Contraintes Techniques
Matériel, logiciels, systèmes d'exploitation, normes technologiques, etc.

* **Front-end** : Application basée sur React et Next.js.
* **Serveur MCP** : Node.js / TypeScript.
* **Format des données** : Markdown brut et YAML pour les métadonnées et schémas.

## 2.2 Contraintes Organisationnelles
Équipe, budget, calendrier, processus de développement, brevets, etc.

* **Calendrier** : Phase de POC rapide pour tester la synchro Live Preview.
* **Méthodologie** : Alignement avec la démarche iSAQB pour les étapes de conception.

## 2.3 Conventions
Principes de codage, normes de documentation, etc.

* **Documentation** : Structure basée sur le template arc42 (Français).
* **Fiches de décision** : Format standardisé ADR (Architecture Decision Record).
